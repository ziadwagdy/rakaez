🌡

DHT22

وصّلناه على Pin 2 وقرأنا الحرارة والرطوبة بمكتبة DHT — قراءة كل 2 ثانية

📳

حساس الاهتزاز

وصّلناه على Pin 3 ويُرسل HIGH عند الاهتزاز — أبسط حساس رقمي في المشروع

📺

Serial Monitor

شاهدنا البيانات تتدفق في الوقت الحقيقي — الخطوة الأولى في رحلة البيانات

🎯

البيانات الحقيقية

قارنّا Demo Mode ببيانات حقيقية — الفرق واضح في الرسم البياني وأنماط التغيير

مفهوم المعمارية
ما هي معمارية النظام؟
معمارية النظام هي المخطط العام الذي يُظهر كل مكوّن وكيف يتواصل مع المكوّنات الأخرى
🖥️

Frontend

ما يراه المستخدم — HTML وCSS وJavaScript يُكوّنون لوحة التحكم التفاعلية في المتصفح

🧠

Backend

عقل النظام — Python و Flask يعالجان الطلبات ويُطبّقان القواعد ويُولّدان التنبيهات

💾

Database

ذاكرة النظام — SQLite يحفظ كل قراءة وكل تنبيه مع وقته بشكل منظّم وقابل للبحث

⚡

Hardware

عيون وآذان النظام — Arduino والحساسات تجمع البيانات من العالم الفيزيائي

تشبيه من الحياة
ما هو API؟ — تشبيه المطعم 🍽
API هو الواجهة التي تسمح للأجزاء المختلفة بالتحدث مع بعضها — مثل النادل في المطعم تماماً
🙋

العميل — Frontend

يجلس في المطعم ويطلب الطعام — مثل المتصفح الذي يطلب بيانات من الـ Backend

🤵

النادل — API

يستقبل الطلب ويُوصّله للمطبخ ويُعيد الطعام — مثل Flask الذي يستقبل الطلبات ويُرجع JSON

👨‍🍳

المطبخ — Backend

يُعدّ الطعام الفعلي ويُخزّن المقادير — مثل Python الذي يُعالج البيانات ويُخزّنها في SQLite

📋

قائمة الطعام — Endpoints

قائمة ما يمكن طلبه — مثل /api/readings و /api/alerts التي يُقدّمها رصد (Rasd)

Backend يتوقف؟

Dashboard لن يحصل على بيانات — سيظهر خطأ في المتصفح — لكن Arduino لا يزال يقرأ

💾

Database تتلف؟

نفقد كل السجل التاريخي للقراءات — لكن النظام قد يستمر في العمل بدون تخزين مؤقتاً

🔌

Arduino ينفصل؟

لا بيانات جديدة — الـ Dashboard سيعرض آخر قراءة محفوظة — تنبيه "فقدان الاتصال"

🌐

الإنترنت ينقطع؟

ngrok يتوقف — لكن النظام يعمل محلياً على localhost — يمكن الوصول من نفس الشبكة
