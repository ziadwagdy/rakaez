🎯 أهداف الجلسة
  • تعلّم جمل الشرط: if / elif / else
  • تعلّم حلقة for وحلقة while
  • فهم متى تستخدم كل نوع من الحلقات
  • بناء لعبة "تخمين الرقم" كاملة

🛠 أنشطة الفصل
  • تطبيق الشرط: برنامج يقول "حار" أو "بارد" حسب درجة الحرارة المُدخلة
  • تطبيق الحلقة: اطبع أرقام ١ إلى ١٠، ثم الأرقام الزوجية فقط
  • مشروع صغير: لعبة تخمين الرقم (الحاسوب يختار، أنت تخمّن)
  • نقاش: كيف يستخدم رصد (Rasd) الشروط للتنبيه؟
🏠 الواجب المنزلي
  • أكمل لعبة التخمين: أضف عداداً للمحاولات وأخبر المستخدم إذا انتهت
  • مسألة: اكتب برنامجاً يطبع جدول الضرب لأي رقم يُدخله المستخدم
  • تحدي: اكتب برنامجاً يتحقق إذا كانت درجة الحرارة "مثالية" (18–25 درجة) أو "شديدة"
👨‍🏫 ملاحظات المدرب
  • الشروط هي نفس مفهوم "القرار" في المخطط الانسيابي — ذكّر الطلاب بذلك
  • لعبة التخمين ممتعة جداً وتُعلّم while loop بشكل طبيعي
  • ابدأ بـ for ثم while — for أبسط مفهوماً للمبتدئين
  • درس التنبيهات في رصد (Rasd) يُجسّد هذه المفاهيم — استخدمه

  • تطبيق الزر: اضغط الزر → يضيء LED، اتركه → يطفئ
  • تعديل: اجعل الـ LED يومض ٣ مرات عند الضغط
  • عرض: كيف يُرسل Arduino بيانات عبر Serial Monitor إلى Python
🏠 الواجب المنزلي
  • إذا توفّر Arduino: جرّب تعديل كود Blink — سرّع أو بطّئ الوميض
  • إذا لم يتوفّر: استخدم Tinkercad (محاكاة مجانية) وجرّب نفس الكود
  • اقرأ: arduino_sketch.ino من كود رصد (Rasd) — ما الذي يفعله؟
  • ارسم: مخطط توصيل DHT22 بـ Arduino (من ورقة البيانات)
👨‍🏫 ملاحظات المدرب
  • جلسة مشتركة: المدرب المشارك يشرح التوصيل، Zeyad يشرح الكود
  • Tinkercad.com يُتيح المحاكاة مجاناً — بديل ممتاز إذا كانت الأجهزة غير كافية
  • اربط بـ رصد (Rasd): "هذا بالضبط ما يفعله Arduino في مشروعنا"
  • الطلاب الذين يُحبون هذه الجلسة هم المرشحون لمسار الإلكترونيات والميكاترونيكس

قاعدة البيانات — SQLite 🗄
Database SQLite رصد (Rasd)
▾
🔄 مراجعة ٢٠د 💡 قواعد البيانات ٦٠د 🛠 استعراض SQLite ١٠٠د 📝 تلخيص ٦٠د
🎯 أهداف الجلسة
  • فهم ما هي قاعدة البيانات ولماذا نحتاجها
  • التعرف على SQLite وكيف تُستخدم في رصد (Rasd)
  • قراءة أوامر SQL الأساسية: SELECT, INSERT, WHERE
  • استعراض البيانات المُخزَّنة في رصد (Rasd) مباشرة
  • فهم جدول الـ readings وجدول الـ alerts
🛠 أنشطة الفصل
  • افتح room_guardian.db باستخدام DB Browser for SQLite
  • استعرض الجداول: ما الحقول؟ ما أنواع البيانات؟
  • نفّذ: SELECT * FROM readings LIMIT 10
  • نفّذ: SELECT * FROM readings WHERE temperature > 25
  • نقاش: لماذا نُخزّن التاريخ (timestamp)؟ ما الفائدة؟
🏠 الواجب المنزلي
  • اكتب استعلاماً SQL يُظهر آخر ٢٠ قراءة فقط
  • اكتب استعلاماً يُحسب متوسط درجة الحرارة (AVG)
  • اكتب في الدفتر: لماذا لا نُخزّن كل شيء في ملف .txt بدل قاعدة بيانات؟
👨‍🏫 ملاحظات المدرب
  • DB Browser for SQLite مجاني وبسيط — ثبّته مسبقاً على الأجهزة
  • رؤية بياناتهم الحقيقية في جدول هو أكثر تجربة مُلموسة يمكن تقديمها
  • SQL يبدو مُخيفاً لكنه بسيط — أسلوب الحياة اليومية يُبسّطه كثيراً
  • في نهاية هذه الجلسة، يجب أن يفهم كل طالب أين تذهب البيانات

👨‍🏫 ملاحظات المدرب

  • اشرح fetch() بتشبيه: "كالنادل الذي يذهب للمطبخ ويعود بالطلب"
  • هذه آخر جلسة قبل التكامل — تأكد أن الجميع يفهم أجزاء المشروع

🎯 أهداف الجلسة
  • تشغيل رصد (Rasd) كاملاً من A إلى Z بدون توقف
  • استخدام ngrok لمشاركة الـ Dashboard عبر الإنترنت
  • تخصيص رصد (Rasd): كل طالب يُضيف لمسته الشخصية
  • اختبار جميع المميزات وإصلاح أي مشاكل
  • تحضير ما سيُقدَّم في العرض النهائي
🛠 أنشطة الفصل
  • تشغيل كامل: Arduino → Python → Flask → SQLite → Dashboard
  • تشغيل ngrok ومشاركة الرابط مع بقية الطلاب
  • اختبر الرابط من الهاتف — هل يعمل؟
  • وقت التخصيص: كل طالب يختار شيئاً يُعدّله (threshold، رسالة، لون)
  • تحضير: ما الذي ستقوله في عرضك النهائي؟ (٣–٥ دقائق)
🏠 الواجب المنزلي
  • شارك رابط رصد (Rasd) مع أحد أفراد أسرتك واحصل على رأيه
  • حضّر: ٣ نقاط ستقولها في عرضك — ما الذي تعلمته؟ ما الذي فعلته؟
  • حضّر: سؤال واحد تودّ طرحه على الفريق أو جواباً تودّ مشاركته
  • فكّر: ماذا تريد أن تتعلم بعد؟ برمجة؟ إلكترونيات؟ تصميم؟
👨‍🏫 ملاحظات المدرب
  • ngrok مُذهل للطلاب — لحظة "مشاركة رابطي مع العالم" لا تُنسى
  • اترك وقتاً كافياً للتخصيص — هذا ما يجعل المشروع "ملكهم"
  • التحضير للعرض مهم: ساعد كل طالب يصياغة ما سيقوله
  • تأكد أن كل طالب لديه شيء يُعرضه ويفخر به في الجلسة الأخيرة
